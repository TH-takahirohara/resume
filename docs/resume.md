### バックエンドアプリケーションの改修（Railsのバージョンアップ、新規インフラ環境構築等）
- 期間
  - 2022年4月 ~ 2022年9月

- プロジェクトの編成
  - チームリーダー: 1名、エンジニア: 3 ~ 4名(担当)

-  使用技術
  - Ruby, Ruby on Rails(バージョン 4系・6系), AWS ECS(Fargate), AWS CodePipeline, AWS CDK(Python)

- 担当業務
  - バックエンドで使われるRuby on Railsのバージョンアップ(バージョン 4系 -> 6系)作業
  - AWS CDKによる新規インフラ環境構築(AWS CodePipelineに関する設定)

- 課題と取り組み
  - Railsのバージョンアップ施策において、バージョンを上げるとエラーが大量に発生するため、何から着手すれば良いかということがチーム内で課題になっていました。そこで、バージョンアップ時において失敗している自動テストを修正することから始めるという方向性を提案しました。これは、例えば特定のクラスのテストがすべて成功している状態であれば、その時点で発生しているエラーはそのクラス以外の箇所に原因がある可能性が高いと判断できるため、原因の特定がしやすくなるだろうと考えたためです。最初の方向性が固まったことで、その後のエラー修正のペースが安定したと考えています。

### 新規機能開発(物件情報の拡充、物件検索機能の追加、ビデオ・音声通話機能の追加)
- 期間
  - 2019年10月 ~ 2020年8月

- プロジェクトの編成
  - リーダー: 1名、エンジニア: 4 ~ 5名(担当)

- 使用技術
  - Swift, UIKit

- 担当業務
  - iOSアプリの開発

- 課題と取り組み
  - iOSアプリでの物件検索機能の作成時に、検索結果を下方にスワイプして取得済みの物件がすべて表示されてから次の物件群の取得を行うと、ローディングによる待ち時間が発生してユーザー体験を損なうため、課題であると考えました。そこで、取得済みのすべての物件が表示される前に、ユーザーのタップ位置を判定して早めに後続の物件群のローディングを行うように工夫するという試みを自主的に行いました。結果として、通常使用する範囲においてはローディングによる待ち時間が発生しないようになり、ユーザー体験を向上させることができました。
